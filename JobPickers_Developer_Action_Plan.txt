  "identifier": {
    "@type": "PropertyValue",

    "value": "jp-12345"
  }

</script>

FIELD HANDLING RULES:
--------------------
- title: Required. Use exact job title, don't stuff keywords.
- description: Required. Full HTML description. Must be unique, not scraped duplicate.
- datePosted: Required. ISO 8601 format. Use original posting date.
- validThrough: Required. Set to 30 days from datePosted if not specified.
- employmentType: Required. Use: FULL_TIME, PART_TIME, CONTRACT, TEMPORARY, INTERN, VOLUNTEER
- hiringOrganization.name: Required. Real company name.
- jobLocation: Required for on-site. Include full address when available.
- jobLocationType: TELECOMMUTE for remote jobs.
- applicantLocationRequirements: Required for remote. Use Country type.
- baseSalary: Recommended but not required. Omit if not available (don't guess).
- directApply: true if application is on your site, false if external link.
- identifier: Unique ID for deduplication.

REMOTE JOBS:
------------
Must include BOTH:
  "jobLocationType": "TELECOMMUTE",
  "applicantLocationRequirements": { "@type": "Country", "name": "USA" }

SALARY MISSING:
---------------
Omit baseSalary entirely. Do NOT include estimated or guessed salaries.
Google penalizes inaccurate salary data.

EXPIRED JOBS:
-------------
Remove JobPosting schema when validThrough date passes.
Google will penalize sites with expired job schema.

EXTERNAL APPLY LINKS:
---------------------
Set "directApply": false when linking to external application page.
Add clear "Apply on [Company]" button with external link icon.

GOOGLE INDEXING API:
--------------------
- Create a Google Cloud service account
- Enable Indexing API
- Submit job URLs immediately after publishing
- Submit remove requests when jobs expire
- Python example:
  ```
  from google.oauth2 import service_account
  import requests
  
  credentials = service_account.Credentials.from_service_account_file(
      'service-account.json',
      scopes=['https://www.googleapis.com/auth/indexing']
  )
  
  endpoint = 'https://indexing.googleapis.com/v3/urlNotifications:publish'
  headers = {'Authorization': f'Bearer {credentials.token}'}
  data = {
      'url': 'https://jobpickers.com/jobs/job-slug',
      'type': 'URL_UPDATED'
  }
  requests.post(endpoint, headers=headers, json=data)
  ```

8. DATA INGESTION ARCHITECTURE
============================================================

This is the #1 priority. Without real jobs, nothing else matters.

RECOMMENDED INGESTION PIPELINE:
-------------------------------

Phase 1: Start with safe, legal sources (Week 1-2)
Phase 2: Scale to RSS and APIs (Week 3-4)
Phase 3: ATS integrations (Month 2-3)

SOURCES TO IMPLEMENT:
---------------------

1. OFFICIAL APIs (Safest, highest quality)
   - US Federal jobs: usajobs.gov API (free)
   - UK Civil Service: APIs available
   - Adzuna API: 500 calls/day free tier
   - Jooble API: Available for partners
   - Reliability: HIGH
   - Legality: HIGH (official APIs)
   - Cost: Free to low

2. RSS FEEDS (Good for specific niches)
   - StackOverflow Jobs RSS (if available)
   - WeWorkRemotely RSS
   - RemoteOK RSS
   - AngelList/Wellfound feeds
   - GitHub Jobs alternatives
   - Company career page RSS feeds
   - Reliability: MEDIUM
   - Legality: MEDIUM (check terms of each source)
   - Cost: Free

3. ATS PLATFORMS (Direct employer feeds)
   - Greenhouse: Job boards API
   - Lever: Postings API
   - Workable: Careers API
   - Ashby: Public API
   - SmartRecruiters: API available
   - Reliability: HIGH
   - Legality: HIGH (employers opt-in)
   - Cost: Free (read public boards)

4. REMOTE JOB APIs
   - RemoteOK API
   - WeWorkRemotely API
   - Remotive API
   - FlexJobs (paid partnership)
   - Reliability: HIGH
   - Legality: HIGH
   - Cost: Varies

5. COMPANY CAREER PAGES (Most complex)
   - Requires custom scraping per company
   - Use libraries like Scrapy, Playwright
   - Must respect robots.txt
   - Must check Terms of Service
   - Reliability: MEDIUM
   - Legality: LOW-MEDIUM (risky without permission)
   - Cost: Infrastructure only

INGESTION PIPELINE STEPS:
-------------------------
1. SOURCE DISCOVERY: Maintain list of active sources with metadata
2. FETCH: Scheduled pulls (cron every 2-4 hours for active sources)
3. PARSE: Extract job data from HTML/JSON/XML
4. NORMALIZE: Convert to standard schema (title, company, location, salary, etc.)
5. CLASSIFY: Auto-categorize (Software, Marketing, Design, etc.)
6. DEDUPLICATE: Hash + fuzzy matching to prevent duplicates
7. ENRICH: Add company info, logos, location normalization
8. SPAM/FRAUD DETECTION: Filter suspicious listings
9. SALARY NORMALIZATION: Convert to USD annual equivalent
10. LOCATION NORMALIZATION: Standardize city/state/country, detect remote
11. REMOTE/HYBRID/ON-SITE DETECTION: Parse description + explicit fields
12. EXPIRY DETECTION: Set 30-day default, check for removed listings
13. ATTRIBUTION: Track source for every job, display "Originally posted on [Source]"
14. PUBLISHING: Make live on site
15. SITEMAP UPDATE: Add new job URLs, remove expired
16. INDEXING API: Notify Google of new/updated jobs

-------------------------------

LEGAL COMPLIANCE NOTES
============================================================

JOB AGGREGATION RISKS:
----------------------
- You CAN list jobs that are publicly posted
- You CANNOT scrape sites that prohibit it in their Terms of Service
- You MUST attribute the original source (link back)
- You MUST respond to takedown requests promptly
- You MUST NOT modify job descriptions to mislead

ATTRIBUTION:
------------
Every job must show: "Originally posted on [Source Name]"
Link to original listing. This protects you legally and
builds trust with users.

DMCA:
-----
Your DMCA page exists - good. Make sure:
- Include physical address for DMCA notices
- Respond within 48 hours to takedown requests
- Keep records of all takedowns

GDPR/CCPA:
----------
Your Privacy Policy mentions these - good. Also need:
- Cookie consent banner (before any tracking)
- Data deletion request process
- Opt-out of sale/sharing (CCPA)
- Data processing agreement if using EU workers' data

TRADEMARK RISK:
---------------
- Do NOT use "Indeed jobs" or "LinkedIn jobs" in your titles
- Do NOT claim to be "the new Indeed" or similar
- Do NOT use company logos without permission
- Your brand "JobPickers" is fine - build your own brand
